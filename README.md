# Ai Arbuz Provider Api

> Self-hosted роутер/агрегатор LLM-ключей с веб-панелью управления.

**Ai Arbuz Provider Api** объединяет ваши внешние API-ключи (OpenAI, Anthropic и любые
OpenAI-совместимые сервисы) в логические **провайдеры**, выдаёт вашим клиентам
собственные ключи с независимыми лимитами, проксирует трафик и прозрачно
конвертирует запросы между форматами OpenAI и Anthropic — всё под вашим контролем,
на вашей инфраструктуре.

---

## Зачем это нужно

- **Один вход для разных провайдеров.** Клиенты обращаются к единому endpoint в
  привычном формате (OpenAI *или* Anthropic), а вы решаете, через какой апстрим
  и каким ключом обслужить запрос.
- **Свои ключи с лимитами.** Выдавайте ключи команде/клиентам, ограничивайте по
  запросам, токенам и сроку действия — независимо от лимитов апстрим-ключей.
- **Отказоустойчивость.** Несколько ключей и провайдеров с автоматическим
  переключением при сбоях и отслеживанием «здоровья».
- **Прозрачная конвертация форматов.** OpenAI ↔ Anthropic: чат, потоковая выдача
  (SSE), tool calling, vision, reasoning/thinking, prompt caching.
- **Контроль и наблюдаемость.** Логи запросов, статистика потребления, маскировка
  секретов, защита от вредоносного поведения апстримов.

## Возможности

| Область | Что умеет |
|---|---|
| **Провайдеры и ключи** | Группировка апстрим-ключей в провайдеры; шифрование секретов at-rest (AES-256-GCM) |
| **Выданные ключи** | CRUD, независимые лимиты (запросы / токены / срок), отзыв без потери настроек |
| **Двойные лимиты** | Раздельный учёт по выданным и по апстрим-ключам |
| **Прокси + конвертация** | OpenAI ↔ Anthropic, включая SSE, tools, vision, thinking, prompt caching |
| **Автопоиск моделей** | Подтягивание списка моделей у апстрима |
| **Проверка ключей** | health-check апстрим-ключей и провайдеров |
| **MCP-обёртка** | Экспонирование MCP-сервера как OpenAI-совместимого сервиса |
| **Prompt-правила** | Модификация входящих запросов по правилам |
| **Response-guard** | Инспекция ответов апстрима на вредоносные паттерны (block / alert / off) |
| **Логи и статистика** | Журнал запросов и агрегированное потребление |
| **Веб-панель** | Управление всем перечисленным (терминальная тема) |

## Архитектура

Чистая слоистая архитектура (Go), зависимости направлены внутрь:

```
domain  →  ports  →  usecase  →  adapter  →  transport (HTTP)
```

- **domain** — сущности и бизнес-правила, без внешних зависимостей.
- **ports** — интерфейсы (точки расширения).
- **usecase** — сценарии (проксирование, лимиты, конвертация, guard).
- **adapter** — реализации портов (SQLite, крипто, HTTP-клиент апстрима, кэш, конвертер).
- **transport** — HTTP: клиентский трафик и API панели.

**Стек:** Go (backend, CGO-free), SQLite (хранилище), Vue 3 + Vite + Pinia
(frontend), Docker для упаковки.

## Быстрый старт (одна команда)

На чистом сервере **Ubuntu/Debian** выполните:

```bash
curl -fsSL https://raw.githubusercontent.com/juushimatsu/ai-arbuz-provider-api/master/install.sh | bash -s -- ваш-домен.com
```

или эквивалентно:

```bash
ARBUZ_DOMAIN=ваш-домен.com bash <(curl -fsSL https://raw.githubusercontent.com/juushimatsu/ai-arbuz-provider-api/master/install.sh)
```

Скрипт [`install.sh`](install.sh) **идемпотентен** (безопасно перезапускать) и сам:

1. ставит зависимости и Docker (с `sudo`, если их нет);
2. клонирует/обновляет репозиторий в `~/ai-arbuz-provider-api`;
3. на первом запуске генерирует секреты в `.env` (`ARBUZ_MASTER_KEY`, пароль
   администратора) и больше их не перезатирает;
4. выбирает свободный порт (по умолчанию `8088`) и биндит его только на
   `127.0.0.1` — без конфликтов с уже работающими сервисами;
5. **скачивает готовый образ** из GHCR и запускает стек через `docker compose`
   (на сервере ничего не компилируется — быстро и без нагрузки на CPU/RAM;
   при необходимости пересборку из исходников можно включить через `ARBUZ_BUILD=1`);
6. проверяет `/api/health` и в конце печатает логин/пароль администратора и
   **готовый блок для вашего обратного прокси** (Caddy):

```caddyfile
ваш-домен.com {
    reverse_proxy 127.0.0.1:<порт>
}
```

Останется вставить этот блок в ваш Caddyfile и перезагрузить Caddy — установщик
**ничего в вашем прокси не меняет**.

> Домен можно не указывать (тогда сервис поднимется на `127.0.0.1:<порт>`, а блок
> прокси нужно будет настроить вручную). Подробности и примеры для Nginx/Traefik —
> в [DEPLOYMENT.md](DEPLOYMENT.md).

## Удаление (одна команда)

Остановить и удалить панель на сервере:

```bash
curl -fsSL https://raw.githubusercontent.com/juushimatsu/ai-arbuz-provider-api/master/uninstall.sh | bash
```

По умолчанию скрипт останавливает контейнеры и удаляет Docker-образ, затем
**спрашивает `[y/N]`** перед удалением данных (SQLite-база с ключами и логами)
и секретов (`.env`). Полное удаление без вопросов:

```bash
curl -fsSL https://raw.githubusercontent.com/juushimatsu/ai-arbuz-provider-api/master/uninstall.sh | bash -s -- --purge
```

Флаг `--keep-data` принудительно сохраняет данные (удаляет только контейнеры и образ).

## Запуск вручную (Docker Compose)

Если предпочитаете без установщика:

```bash
git clone -b master https://github.com/juushimatsu/ai-arbuz-provider-api.git
cd ai-arbuz-provider-api
cp .env.example .env
#   Задайте надёжные ARBUZ_ADMIN_PASSWORD и ARBUZ_MASTER_KEY (приложение
#   отказывается стартовать с пустым/слабым паролем)
docker compose up -d
```

Приложение слушает локальный порт и предназначено для работы **за обратным
прокси** (TLS, домен). Порт настраивается через `ARBUZ_HOST_PORT` в `.env`.

## Конфигурация

Все настройки — через переменные окружения (12-factor). Ключевые:

| Переменная | Назначение |
|---|---|
| `ARBUZ_ADMIN_PASSWORD` | Пароль администратора панели (**обязателен**, без дефолта) |
| `ARBUZ_MASTER_KEY` | Мастер-ключ шифрования секретов at-rest (**обязателен**) |
| `ARBUZ_LISTEN` | Адрес/порт прослушивания (внутренний) |
| `ARBUZ_HOST_PORT` | Публикуемый на хосте порт (Docker Compose) |
| `ARBUZ_PUBLIC_URL` | Публичный URL роутера (ваш домен) |
| `ARBUZ_GUARD_MODE` | Режим response-guard: `block` / `alert` / `off` |
| `ARBUZ_LIMIT_FAIL` | Поведение лимитера при ошибке учёта: `closed` / `open` |
| `ARBUZ_CACHE_ENABLED`, `ARBUZ_CACHE_TTL_SECONDS` | Кэш ответов |

Полный список и значения по умолчанию — в `.env.example`.

## Безопасность

- Секреты апстримов шифруются at-rest (AES-256-GCM, случайный nonce).
- Пароль администратора хранится как bcrypt-хэш; слабые/placeholder-пароли
  отклоняются на старте.
- Секреты маскируются в логах.
- Лимитер по умолчанию работает в режиме fail-closed.
- Приложение рассчитано на работу за TLS-терминирующим обратным прокси.
